# Builder Tanda 1 вЂ” fidelidad inmediata (paquete del mockup + backbone UX)

## Contexto

El builder v2 (`26-07-18-catalogo-v2-builder-visual.md`, F01вЂ“F13 hechas, E2E verde) dejГі motor + estilo por secciГіn + tema + editor visual completos, pero la rГ©plica del mockup real `C:/Users/NicolГЎsChaima/Downloads/landing_idol (1).html` en la tienda prueba llegГі solo a ~85%: falta el segundo color de marca (el fondo pГєrpura+dorado y la palabra "ENRIQUECER" dorada son imposibles con un solo hue), el precio destacado del hero, el ticker en marquee sobre el nav, el nav configurable y las estadГ­sticas sin cards. AdemГЎs el editor todavГ­a se siente "formulario de admin": cada mutaciГіn recarga el iframe entero, hay botones "Guardar X" por pestaГ±a y el reorden es a botonazos в†‘в†“.

La investigaciГіn multi-agente `.scratch/page-builder/mapa-potencial-editores.md` (matriz de capacidades vs Wix/Webflow/Shopify/Framer + В§3 evoluciones de arquitectura + В§5 roadmap por tandas + В§6 lo-que-NUNCA) es el insumo del grill вЂ” el usuario dio la directiva de construir la **Tanda 1** ("fidelidad inmediata") directo a implementaciГіn. Este plan cubre: la evoluciГіn A (`colorAcento`, la llave maestra), el paquete de estilo/widgets del mockup, el confetti post-pago, y el backbone de UX del editor (patch en vivo + auto-save + dnd). Criterio de cierre: **el mockup replicado al 100%** (F12). Las decisiones tГЎcticas de este plan son REVISABLES pero no bloquean el arranque.

## Decisiones

Tomadas del mapa (В§5 Tanda 1, aprobado por directiva del usuario) o por el planner (marcadas REVISABLE):

- **D1 вЂ” `colorAcento` es la ГљNICA superficie nueva de hex, y vive en `Tenant`, no en el documento.** Columna `colorAcento String?` nullable (migraciГіn no-op, schema-guardian), tratamiento espejo de `colorPrimario`: `esHex` valida в†’ `generarEscalaColor` deriva 10 tonos в†’ `overrideDesdeBranding` agrega `colors.acento` al theme (CSS vars `--mantine-color-acento-N`). Sin acento vГЎlido в‡’ los esquemas/tonos `acento*` **degradan a la escala de marca** (el doc parsea y renderiza legible siempre вЂ” nunca opciГіn muda ni error). RazГіn: mapa В§3.A вЂ” todo lo visual de la tanda depende de esto; es duplicar maquinaria existente.
- **D2 вЂ” El campo colorAcento se edita en el EDITOR (panel Tema), NO en el admin.** `src/pages/admin/*` y `src/components/admin/*` son ZONA PROHIBIDA (sesiГіn del usuario). Procedure tRPC nuevo en el router `pagebuilder` (p.ej. `setColorAcento`) gated por la misma membresГ­a server-side (`puedoEditar`), input hex validado con `esHex`, escribe `Tenant.colorAcento`. Como el acento vive fuera del documento, el MCP NO gana tool para esto (I12 del plan padre: MCP sin efectos fuera del documento). **Pendiente ANOTADO para la sesiГіn del usuario**: input de colorAcento en `admin/configuracion` junto a colorPrimario.
- **D3 (REVISABLE) вЂ” Bicolor como rama nueva del discriminated union**: `{ tipo:"bicolor", colorA, colorB: enum TONOS_FONDO, direccion: enum ["vertical","horizontal","diagonal"], mezcla: enum ["dura","suave"] }`.strict(). `TONOS_FONDO` = enum curado de tonos con texto emparejado (subset tipo `marca | marca_suave | marca_profundo | acento | acento_suave | acento_profundo | tinta | superficie`). Mezcla dura = hard-stop 50%, suave = gradiente entre A y B. **Legibilidad por construcciГіn**: el color de texto de la secciГіn se empareja con `colorA` (el tono dominante donde se asienta el contenido); el implementer con `frontend-reviewer` afinan el mapa de emparejamiento si una combinaciГіn queda ilegible. Todo en `estiloSeccionACss()` puro (SSR+cliente sin mismatch).
- **D4 (REVISABLE) вЂ” `anchoFondo` como enum nuevo en `EstiloSeccion`** (p.ej. `["completo","contenido"]`), separado del `ancho` existente (que sigue gobernando el Container del contenido). **El default DEBE reproducir el comportamiento actual del wrapper** (I-H): el implementer verifica primero cГіmo pinta hoy `seccion-wrapper.tsx` y fija el default en no-op. Si hoy el fondo ya es full-bleed siempre, `anchoFondo` default `"completo"` y la opciГіn nueva es acotarlo.
- **D5 (REVISABLE) вЂ” `tituloAcento` puente (pre-runs de Tanda 2)**: en `hero`, campo opcional `tituloAcento: { palabra: в‰¤40, estilo: enum ["acento","resaltado","gradiente"] }`.strict(). El render matchea la **primera ocurrencia case-insensitive** de `palabra` dentro de `titulo` y la envuelve en un `<span>` con estilo por token: `acento` = color de la escala acento; `resaltado` = destacador como **background-image del propio span** (lecciГіn en memoria del proyecto вЂ” nunca capa aparte con z-index); `gradiente` = background-clip:text con tokens marca/acento. Sin match в‡’ tГ­tulo normal (degradaciГіn silenciosa). JamГЎs HTML del tenant. Cuando llegue el sistema de runs (Tanda 2, mapa В§3.B), lo absorbe con migrate-on-read.
- **D6 (REVISABLE) вЂ” Resto del hero puente, todo aditivo sin v-bump** (campos opcionales/default в‡’ docs v2 parsean igual): `destacado: { texto в‰¤24, nota в‰¤120 }` (el "$3.000 + nota"), `ctaSecundarioEstilo: enum ["boton","enlace"] default "boton"`, `mostrarConfianza: boolean default true` (los trust badges hoy hardcodeados ganan toggle; `mostrarBadgeSorteo` YA existe вЂ” no duplicar). `efectoTitulo: enum ["ninguno","revelar_palabras","gradiente_animado"] default "ninguno"`: reveal por palabra vГ­a `AnimarItem` sobre spans (SSR sale con el texto VISIBLE вЂ” I-D вЂ” y el stagger solo arma client-side post-hidrataciГіn), gradiente animado reusa el keyframe holo; reduced-motion в‡’ estГЎtico final (I-B). - **D7 (REVISABLE) вЂ” Cinta sobre el nav = `aviso_barra` v2** (ya renderiza arriba de todo, no hace falta overlay nuevo): `mensajes: array(в‰¤40 chars) 1вЂ“5`, `modo: enum ["estatico","rotacion","marquee"] default "estatico"` (marquee = ticker del mockup, CSS puro, pausa en hover, reduced-motion в‡’ estГЎtico), `esquema: enum ESQUEMAS_FONDO default` el look actual, `mostrarCountdown: boolean default false` (chip del sorteo activo, dato server-side ya resuelto вЂ” reusa `use-sorteo-activo`/countdown chip). **v-bump v1в†’v2** con migrate-on-read trivial `texto в†’ mensajes:[texto]` que conserva el look actual (I-H). `enlaceUrl/enlaceTexto/descartable` se conservan.
- **D8 (REVISABLE) вЂ” Nav auto-derivado**: el envelope de secciГіn (`nodoSeccion()`) gana campo opcional `nav: { incluir: boolean, etiqueta?: в‰¤20 }`.strict(). `storefront-layout` deriva el nav del documento published: secciones con `nav.incluir=true` en orden del doc, etiqueta explГ­cita o la del mapa default `tipoв†’etiqueta` (catalogoв†’"CatГЎlogo", sorteo/bloque_ticket_promoв†’"Sorteo", perfil_autoraв†’"Autora", garantias_sorteoв†’"Bases", como_funcionaв†’"CГіmo funciona", faqв†’"Preguntas"вЂ¦). **Si NINGUNA secciГіn configura `nav` в‡’ el nav actual exacto (CatГЎlogo/Sorteo/CГіmo funciona) вЂ” no-op.** Anclas: `SeccionWrapper` ya emite `id` DOM por nodo; ademГЎs emite **ancla semГЎntica por tipo** (la primera secciГіn de cada tipo gana `id` semГЎntico estable вЂ” hoy ya pasa con `#catalogo`/`#sorteo`/`#como-funciona`, se generaliza). `CTA_ANCLAS` se amplГ­a con las anclas semГЎnticas nuevas (p.ej. `autora`, `bases`, `como-funciona`) вЂ” aditivo, docs viejos parsean.
- **D9 вЂ” Alturas**: `EstiloSeccion` gana `altoMin: enum ["auto","media","pantalla"] default "auto"` (pantalla = `min-height: 100svh` вЂ” svh para mobile) y `alinearVertical: enum ["arriba","centro","abajo"] default "arriba"`. `min-height` no se anima (CLS irrelevante, I-C intacto).
- **D10 вЂ” `estadisticas` estilo simple**: `estiloVisual: enum ["cards","simple"] default "cards"` (actual). `simple` = cifra grande + etiqueta, sin `ThemeIcon` ni contenedor card вЂ” las stats limpias del mockup. Count-up se conserva.
- **D11 (REVISABLE) вЂ” MГЎscaras de forma**: `imagen_destacada` gana `forma: enum ["ninguna","circulo","blob","arco","ticket"] default "ninguna"` в†’ clip-paths CURADOS nuestros (blob = path fijo, ticket reusa el motivo `perforacion`).
